The Decision Workflow Lifecycle

From business logic to auditable execution in six steps.

Architecture: Ingest → Design → Compile → Execute → Audit

1. Ingest

Data from your systems normalized via RDI or adapters

2. Objects

Typed, validated inputs and outputs

3. Design

Visual editor for business logic

4. Compile

Deterministic bytecode, no runtime interpretation

5. Execute

Redis Streams orchestrate immutable decisions

6. Audit

Results to RedisJSON, dashboards via Pub/Sub

Data Integration

Your decision inputs live in databases, data warehouses, or event streams. Ferrum bridges the gap.

Connectors to Postgres, MySQL, Oracle, Mongo. Real-time sync. One configuration.

Option B: Lightweight Adapters

REST API translators, scheduled sync, event stream consumers (Kafka, Kinesis).

Object Library & Type Safety

A decision requires inputs. Those inputs have structure. We enforce it.

Input: LoanApplication
  - credit_score: 300–850 (required)
  - annual_income: USD (required)
  - loan_amount: USD (required)

Output: UnderwritingDecision
  - decision: approved | denied | escalate
  - rationale: text (required)
  - audit_timestamp: ISO 8601 (auto)

Compilation

Flows are not interpreted at runtime. They are compiled.

  • Validates all paths (no dead code, no missing branches)
  • Optimizes execution (reorder steps, cache queries)
  • Generates audit metadata (which rule fired, why)
  • Creates deterministic hashes (same flow = same hash)
  • Outputs bytecode or optimized dependency graph

Execution on Redis Streams

When a decision is triggered, Redis Streams orchestrate it.

  • Trigger: Event written to Redis Stream
  • Consumer Group: Guarantees at-least-once processing
  • State Machine: Each step writes state (immutable log)
  • Queries: Flow steps query RedisJSON, invoke AI
  • Output: Decision stored in Streams and RedisJSON

Real-Time UI & Artifact Generation

Decisions Generate Evidence

Decision ID, flow version, input snapshot, rule trace, model inference, confidence score, explainability, timestamp & duration.

AI Integration: Where It Fits (And Where It Doesn't)

AI goes here:

  • Classification: "Is this fraud?"
  • Extraction: "What is the customer's intent?"
  • Ranking: "Rank tickets by urgency"
  • Recommendation: "What product to suggest?"
  • Content Moderation: "Is this policy-violating?"

AI does NOT go here:

  • Writing business logic: Encode rules instead
  • Making binary decisions: Use rules + AI confidence
  • Generating audit-critical output: Generate from rules

Your business logic decides. AI informs. The combination is auditable.
